GCP VM Internals
What Lives Inside the VM
All components run directly on the Compute Engine VM. No Kubernetes — lightweight, simple, cost-effective for retail deployment.
🌐

Nginx Reverse Proxy (Port 80 / 443)

Terminates HTTPS. Routes /api/* → NestJS :8080. Routes / → React static files. Handles SSL termination.

GATEWAY
⚙️

NestJS Backend — Port 8080

Core API server + Socket.IO WebSocket server. Runs as a Node.js process (or inside Docker). Connects to PostgreSQL via Prisma ORM.

BACKEND
📡

Socket.IO Gateway — Port 8080 (WS Upgrade)

WebSocket server co-hosted with HTTP on port 8080. Upgrades HTTP connections. Room-based pub/sub for ticket status events.

REALTIME
⚛️

React Frontend — Static Files

Built with Vite and served as static HTML/JS/CSS by Nginx from /var/www/tickeflow/. Three entry points: index.html (customer), merchant.html (staff), tv.html (display).

FRONTEND
🐘

PostgreSQL — Port 5432

Primary relational database. Managed via Prisma ORM with schema migrations. Stores all tickets, customers, orgs, branches, statuses, and push subscriptions.

DATABASE
🔔

Web Push Service (VAPID)

Browser Push Notifications via web-push library. VAPID key pair configured in .env. Subscriptions stored in PushSubscription table.

Real-Time Layer
WebSocket Events — Socket.IO
All events are namespaced by ticket room (ticket:<ticketId>). Clients join their room on connection using the DQR token.
CLIENT→SERVER

joinRoom

Customer joins their ticket room. Payload: { ticketId, token }. Server validates token before allowing join.

SERVER→CLIENT

ticketStatusUpdated

Broadcast when staff changes status. Payload: { ticketId, status, statusName, timestamp }. Triggers UI update + push notification.

SERVER→CLIENT

ticketClosed

Emitted when order is collected. Payload: { ticketId, closedAt }. Frontend auto-dismisses session and prompts new DQR.

SERVER→TV

counterUpdate

Sent to TV display room when next ticket is called. Payload: { ticketNumber, departmentId, counter }. Drives the in-store display board.

AUTO-TRIGGER

Device Scan → Auto Status

When staff scans DQR → POST /tickets/scan → NestJS auto-advances ticket to device's defaultStatus → emits ticketStatusUpdated over WebSocket — all in one server-side chain.
