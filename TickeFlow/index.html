Project Overview

Meet TickeFlow — The Digital Queue Revolution

TickeFlow replaces physical queues with a fully digital experience. Customers scan a static QR code at the counter, register once with their mobile number, and instantly receive a unique Dynamic QR (DQR) ticket.

Staff scan the DQR to begin processing. Real-time status updates flow back to the customer's browser session via WebSockets — no app download required.

Multiple counters (Butchery + Fishery) are managed simultaneously in a swipeable card dashboard, all from the customer's phone browser.

Features
Everything You Need, Nothing You Don't
Purpose-built for retail counters — every feature was designed to solve a real problem in the butchery, fishery, and meat cutting workflow.
📲

Static Store QR Code

Each counter has a permanent QR code. Scanning opens the web app instantly — no app install required. Works on any smartphone browser.

🎯

Dynamic QR (DQR) Tickets

Each session generates a unique, device-bound, session-scoped DQR. Tokens are cryptographically signed and expire automatically after collection.

⚡

Real-Time Status Updates

WebSocket-powered status push. When staff updates an order, customers see it on their screen in under 100ms — no polling, no refresh needed.

👤

One-Time Registration

Customers register once (name, mobile, email). On future visits, OTP verification auto-fills their details — frictionless repeat experience.

↔️

Multi-Session Dashboard

Customers can hold multiple DQRs simultaneously (e.g., Butchery + Fishery). A swipeable card interface shows the live status of each ticket.

🔔

Push Notifications

Browser push notifications alert customers even when they're not actively on the page. "Your order is ready!" delivered instantly.

🖥️

TV Display Mode

A TV dashboard view shows the current call number at the counter for customers in-store — bridging digital and physical signage.

📊

Admin Dashboard

Full management suite: store config, staff management, live ticket monitoring, status override, reports, and analytics with trend visualization.

🔄

Automatic Session Closure

When a customer collects their order, the session closes automatically. The DQR is invalidated and a new one must be generated for the next order.

From a standalone executable for small stores to containerized cloud deployments for enterprise retail chains — TickeFlow adapts to your infrastructure.

Standalone Executable

Security & Privacy
Built Secure From the Ground Up
🔐

Device-Bound DQR Tokens

Each DQR is cryptographically bound to the customer's device fingerprint. Tokens cannot be transferred or replayed on another device.

📱

Mobile OTP Authentication

Phone number verified via OTP at registration and on each new session. Prevents fake registrations and unauthorized access.

⏱️

Auto Token Expiration

JWT tokens and DQR sessions expire automatically. Expired tokens are rejected server-side. No persistent login vulnerabilities.

🛡️

Session Hijacking Prevention

Sessions are tied to store, device, and user. Cross-store access is blocked. Each DQR room in Socket.IO is isolated and authenticated.

🔒

HTTPS / TLS Enforced

All communication between client and server is encrypted in transit. Certificate validation enforced in production builds.

👁️

Data Minimization

Only essential PII collected (name, mobile). Data is scoped to store. GDPR-aligned consent flow on first registration.
